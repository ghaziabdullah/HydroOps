============================================================
1) GLOBAL APP SHELL (APPLIES TO ALL PAGES)
============================================================

1.1 Layout
- App uses a fixed left sidebar + top header (topbar) + scrollable main content area.
- Layout grid:
  - Sidebar: fixed, ~260px width desktop; collapsible to icon-only (~76px) using a collapse button near the logo.
  - Main: full height, with topbar row and content below.
  - Content uses card-based sections with generous padding and rounded corners.

1.2 Sidebar (Left Navigation)
- Top-left brand:
  - HydroOps logo icon + “HydroOps” text.
  - Collapse/expand button to the right of brand (double-chevron style).
- Navigation items (vertical list):
  1) Overview
  2) Hostels
  3) Units
  4) Alerts
  5) Water Quality
  6) Reports & Insights
  7) AI & Forecasting
  8) Maintenance
- Bottom section:
  - Settings item pinned near bottom.
- Nav item styling:
  - Each nav item has an icon at left.
  - Active item uses highlighted pill/rounded background and brighter text.
  - Inactive items are muted.
- Hover: subtle background + slightly brighter icon/text.

1.3 Topbar (Header)
- Left side:
  - Current page title or a dropdown-like title:
    - Example: “Campus Overview Dashboard” (with a small caret down).
    - Example: “Main Campus Hostels List / Comparison”
    - Example: “Main Campus Hostel Detail Dashboard”
  - On light theme campus page shows “NUST H-12 Campus” in header area; include campus name in title area (optional).
- Center-left controls:
  - Date-range selector group: “24h / 7d / 30d” with “24h” active in the screenshots.
  - A small rounded “24h” button appears to the right (acts like apply/current selection indicator).
- Center-right:
  - Search input with placeholder: “Search hostel, unit, sensor”
- Right side:
  - Notification bell icon with badge count (e.g., red badge “12”).
  - User pill/menu: avatar circle + role/name “operator” + dropdown caret.

1.4 Common Cards / Surfaces
- All content is inside “cards” with:
  - Rounded corners (large radius ~16px).
  - Subtle border.
  - Soft shadow (light theme) or soft glow/contrast (dark theme).
  - Headings are medium-weight; key numbers large.

1.5 Typography
- Modern sans-serif (Bootstrap default ok, or Inter).
- KPI numbers large (32–40px).
- Labels small/medium (12–14px) with muted color.
- Table text medium (14px).

1.6 Buttons / Pills / Badges
- Pills used for:
  - Status: Optimal / Green, Warning / Orange, Critical / Red.
  - Filter chips at top of tables.
- “View” action is a small outlined button (rightmost table column).
- Severity pills appear inside alert lists.

1.7 Charts (Chart.js)
- Line charts:
  - 2 series used: “Inlet” and “Sum of Hostels”.
  - Series colors (dark theme): cyan/teal for Inlet; purple/violet for Sum.
  - Filled area under the line (soft gradient).
  - Smooth curves.
- Chart controls:
  - Top-right inside chart card: a small range dropdown: “24h / 7d / 30d”.
- Chart annotation:
  - A vertical highlighted region labeled “Suspected Leak Window” (translucent red/pink overlay).
- Bar chart:
  - “Top Consuming Hostels Today” bar chart with a cyan/purple gradient bars.
- Donut chart:
  - “Usage Distribution by Hostel” donut with multiple bright segments.

1.8 Responsiveness (Required)
- Desktop: multi-column cards (KPIs in one row, charts with right-side alert stack).
- Mobile:
  - Sidebar collapses to icons or becomes offcanvas.
  - KPI cards become horizontally scrollable or wrap 1-per-row.
  - Charts full width, right-side panels move below.

2) THEME SYSTEM (DARK + LIGHT)
============================================================

2.1 Theme Toggle Behavior (Global)
- Default theme: DARK.
- Toggle located in topbar (you can add it near search or user menu).
- Persist selection in localStorage key: "hydroops_theme" = "dark" | "light".
- Apply theme via:
  - <html data-theme="dark"> or <body data-bs-theme="dark"> style approach.
  - CSS variables set on [data-theme].

2.2 Dark Theme (Inferred from images 1–3)
- Overall background: deep navy/charcoal gradient.
- Sidebar background: slightly darker than content.
- Card background: dark slate with subtle gradient.
- Borders: low-contrast, cool-toned.
- Text: near-white; muted text gray-blue.
- Accent primary: teal/cyan glow for active nav and some highlights.
- Secondary accent: violet/purple for second chart series.
- Status colors:
  - Green: success/optimal.
  - Orange: warning.
  - Red: critical.

2.3 Light Theme (Derived from image4; apply “not plain white”)
Light theme must feel “soft, modern, slightly bluish” not pure white:
- Base background: very light cool gray/blue (#F3F7FB to #F7FAFF range).
- Sidebar background: slightly different tint from main background (a bit more blue, very subtle).
- Cards:
  - Background: off-white with slight blue tint (#F8FBFF) + subtle shadow.
  - Border: light gray-blue (#E1EAF3).
- Text:
  - Primary text: near-black but softened (#0F172A-ish).
  - Muted text: slate gray (#64748B-ish).
- Charts:
  - Gridlines visible but very faint.
  - Same series colors as dark theme but slightly desaturated.
- Alert highlight (“Suspected Leak Window”):
  - Keep soft red translucent overlay; ensure text is readable on light background.
- Buttons/pills:
  - Use filled soft pills for statuses; keep contrast high enough for accessibility.
- Overall look:
  - Use soft shadows, not harsh borders.
  - Keep spacing and rounded corners identical to dark theme for consistency.

RECOMMENDED LIGHT THEME TOKENS (CSS variables)
- --bg: #F3F7FB
- --sidebar-bg: #EEF4FA
- --surface: #F8FBFF
- --surface-2: #FFFFFF
- --border: #E1EAF3
- --text: #0F172A
- --text-muted: #64748B
- --primary: #14B8A6 (teal)
- --secondary: #8B5CF6 (violet)
- --info: #38BDF8
- --success: #16A34A
- --warning: #F59E0B
- --danger: #EF4444
